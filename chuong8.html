Power Pivot & Data Model

Nâng cấp Pivot Table lên tầm cao mới: liên kết nhiều bảng dữ liệu, viết công thức DAX, và xử lý hàng triệu dòng.

⚡ 8.1 Power Pivot là gì?

Power Pivot là add-in miễn phí trong Excel (có sẵn từ Excel 2016+) mở rộng khả năng của Pivot Table thông thường.

📊 So sánh Pivot Table thường vs Power Pivot

Nguồn dữ liệu1 bảng duy nhấtNhiều bảng liên kết

RelationshipsKhôngCó (giống SQL)

📌
Khi nào dùng Power Pivot? Khi dữ liệu có nhiều bảng cần liên kết (ví dụ: Orders + Products + Customers), hoặc cần tính toán phức tạp hơn Calculated Field.

🔗 8.2 Data Model & Relationships

Data Model là nơi chứa tất cả bảng dữ liệu và mối quan hệ giữa chúng (giống database).

Tạo Relationships

Chuyển 3 bảng thành Table (Ctrl + T) → đặt tên: tblOrders, tblProducts, tblCustomers

Tab Data → Relationships → New...

Relationship 1: tblOrders[ProductID] → tblProducts[ProductID]

Relationship 2: tblOrders[CustomerID] → tblCustomers[CustomerID]

Tạo PivotTable: Insert → PivotTable → tick "Add this data to the Data Model"

💡
Kết quả: Bây giờ bạn có thể kéo ProductName từ bảng Products, CustomerName từ bảng Customers vào cùng 1 Pivot Table! Không cần VLOOKUP.

📐 8.3 DAX cơ bản

DAX (Data Analysis Expressions) là ngôn ngữ công thức của Power Pivot, mạnh hơn Calculated Field rất nhiều.

📊 Các hàm DAX cơ bản

Hàm DAXChức năngVí dụ
SUM()Tổng cộng=SUM(Orders[Quantity])
AVERAGE()Trung bình=AVERAGE(Orders[Quantity])
COUNT() / COUNTA()Đếm=COUNT(Orders[OrderID])
DISTINCTCOUNT()Đếm giá trị duy nhất=DISTINCTCOUNT(Orders[CustomerID])
RELATED()Lấy giá trị từ bảng liên kết=RELATED(Products[UnitPrice])
CALCULATE()Tính toán với filter context=CALCULATE(SUM(Orders[Revenue]), Products[Category]="Electronics")
SUMX()Tính tổng từng dòng rồi cộng lại=SUMX(Orders, Orders[Quantity] * RELATED(Products[UnitPrice]))

💡 Ví dụ: Tính Revenue bằng DAX

Thay vì thêm cột Revenue trong bảng Orders, dùng Measure DAX:

Total Revenue := SUMX(Orders, Orders[Quantity] * RELATED(Products[UnitPrice]))

Measure này tính: cho mỗi dòng Orders, lấy Quantity × UnitPrice (từ bảng Products qua relationship), rồi tổng tất cả.

⚠️
CALCULATE vs SUMX: CALCULATE thay đổi filter context (lọc trước rồi tính). SUMX lặp qua từng dòng rồi tính. Kết hợp cả 2 tạo ra tính toán rất mạnh.

🎯 8.4 Bài tập thực hành

📝 Bài tập 8.1 – Liên kết 3 bảng

Yêu cầu:

  1. Tạo 3 bảng: Orders (20 dòng), Products (5 sản phẩm), Customers (5 khách hàng)
  2. Chuyển thành Table, đặt tên phù hợp
  3. Tạo Relationships: Orders↔Products (ProductID), Orders↔Customers (CustomerID)
  4. Tạo Pivot Table từ Data Model
  5. Kéo: Rows = ProductName (bảng Products), Columns = City (bảng Customers), Values = Sum of Quantity
  6. Kết quả: Bảng crosstab sản phẩm × thành phố — không cần VLOOKUP!

✅ Key steps:

Tạo 3 Tables trên 3 sheets khác nhau

Data → Relationships → New → tạo 2 relationships

Insert → PivotTable → tick "Add this data to the Data Model"

Trong PivotTable Fields: chọn "ALL" ở phía trên → thấy cả 3 bảng

Expand bảng Products → kéo ProductName → Rows

Expand bảng Customers → kéo City → Columns

Expand bảng Orders → kéo Quantity → Values
